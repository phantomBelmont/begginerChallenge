 🌙 星の夜のミニゲームコレクション：没入感とレスポンシブデザインの最適化

 🚀 概要
このプロジェクトでは、神秘的な「星の夜」をテーマにした2つのインタラクティブなミニゲーム（シャッフルゲーム、フリップカードゲーム）を実装しました。
単なる機能の実装にとどまらず、「スクロールなしで画面に収まる完璧なレイアウト」や「デバイスに応じた自動調整」、「画面遷移時の滑らかな演出」など、ユーザー体験（UX）を最大化するための技術的工夫を凝らしています。

---

 ✨ 主な実装内容と技術的工夫

 🎮 1. 2種類のミニゲーム
   シャッフルゲーム: 自然、動物、ストーリーの3カテゴリから、Fisher-Yatesアルゴリズムを用いた真のランダムな組み合わせを生成。結果を視覚的に見やすく表示。
   フリップカードゲーム: カテゴリ別に整理されたカードを、クリックで表裏切り替え可能。全カードを消費すると自動でシャッフルされ、無限に遊べるループ構造を実装。

 🌌 2. 没入感のあるビジュアル演出
   動的星空背景: CSSのグラデーションとアニメーションのみで、星がゆっくり移動し、瞬きする「動く夜空」を表現。
   連動する月: メニュー画面では神秘的な月を表示し、ゲーム開始と同時に自然に消える演出により、画面ごとの雰囲気を切り替えました。
   ダークモードUI: 赤黒い発光エフェクトとダークグレーの背景を採用し、宇宙空間にいるような没入感を演出。

 📱 3. レスポンシブデザインとレイアウトの最適化（最大の特徴）
このプロジェクトの最大の見どころは、「あらゆるデバイスでスクロールなしで綺麗に収まる」という課題を解決したことです。

   Flexboxの高度な活用:
       `flex: 1 1 50vh`（伸び縮み自由）と `flex: 0 0 auto`（固定）を組み合わせ、カードの高さとボタンの高さを動的に調整。
       画面の高さが変わっても、カードとボタンが重なることなく、常に画面内に収まる「賢いレイアウト」を実現しました。
   デバイス別調整:
       スマホ、タブレット、PCそれぞれの画面サイズに合わせて、文字サイズや余白、ボタン幅を自動調整。
       `@media`クエリを活用し、縦向き・横向き問わず見やすい表示を確保。
   スクロール制御の工夫:
       ゲーム画面が開いている間だけJavaScriptでスクロールを一時停止し、カードが画面からはみ出るのを防止。
       メニューや他の画面では自由にスクロールできるようにし、ユーザーの操作感を損なわない設計にしました。

 🛠️ 4. コードの質と保守性
   モジュール化: 画面切り替え、ゲームロジック、スタイルをファイルごとに分離し、メンテナンス性を向上。
   CSSのベストプラクティス:
       `box-sizing: border-box`の統一で、パディングを含めた正確なサイズ計算を実現。
       競合を解消するための戦略的なスタイル定義により、意図した通りに要素が表示されるよう調整。

---

 🧩 解決した技術的課題

   「画面からはみ出る」問題の解決:
    当初、カードとボタンを合わせた高さが画面（100vh）を超え、スクロールが発生していました。
    Flexboxの特性を深く理解し、カードが「伸び縮み」して余白を埋める仕組みを導入することで、スクロールバーを一切出さずに全ての要素を収容するレイアウトを完成させました。

   「不要な要素が残る」問題の解決:
    画面切り替え時に、前の画面の要素（月やボタン）が隠れきらずに残って表示される現象が発生。
    CSSの優先順位（Specificity）を調整し、非表示クラス（`.hidden`）を確実に適用させることで、きれいな画面遷移を実現しました。

   レスポンシブの崩れ:
    画面サイズが変わると文字が切れたり、ボタンが押しづらくなったりする問題を、`vh`（画面の高さ）と`%`（親要素の割合）の使い分け、および`@media`クエリによる微調整で解消。
